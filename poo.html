OOP : abreviatura de la Programación Orientada a Objetos

Consideraciones

Imaginemos le siguiente supuesto donde deseamos de dar de alta dos personas:
$nombre = 'Carlos'; $edad = 23; $pais = 'España';
Si deseamos otra persona:
$nombre2 = 'María'; $edad2 = 22; $pais3 = 'España';
Como observamos tenemos que copiar o reproducir las mismas variables en cada una de las personas. Si tenemos que copiar en programación estamos haciendo algo mal.

Una posible solución sería la POO

En el ejemplo de arriba tenemos un patrón, usuarios con nombre, edad y país.

  1. Lo que necesitamos en una plantilla donde podamos almacenar estos datos, en definitiva una clase.
    //creación de una clase class Persona{   public $nombre;   public $edad;   public $pais; }
  2. La forma de cargar los datos sería la siguiente.
    //cargamos los datos //datos de carlos $carlos = new Persona(); $carlos->nombre = "Carlos"; $carlos->edad = 23; $carlos->pais = "España"; //datos de María $maria = new Persona(); $maria->nombre = "María"; $maria->edad = 22; $maria->pais = "España"; echo $maria->nombre.' tiene '.$maria->edad.' años y vive en '.$maria->pais

Funciones

También es posible agregar funciones / métodos a nuestras clases.

class Persona{   public $nombre;   public $edad;   public $pais;
//solución   public function mostrarInformacion(){     echo "Hola, te saludo";   }
}
Y cómo lo utilizamos:
//Utilización de un método tras instanciación de un objeto. $carlos = new Persona(); $carlos->nombre = "Carlos"; $carlos->edad = 23; $carlos->pais = "España"; $carlos->obtenerInformacion();
